## Концепция

"Разерешение" представляет собой разрешение на определенное действие.
Разрешение может быть безусловным или условным.
Разрешения имеют иерархию. Для удобства работы.
Разрешение может содержать дочерние разрешения, которые автоматически выполняются при выполнении текущего разрешения.
Разрешение может содержать дополняющие разрешения, выполнение каждого из которых автоматически выполняет текущее разрешение.
Условное разрешение не должно само лезть в др.сервисы(бд) за нужной информацией, вся информация для проверки должна предоставляться внешним кодом.


Пример:
 Разрешение на просмотр данных. Содержит:
  - условие: нет
  - дочерние разрешения:
    - разрешение на просмотр определенного сегмента данных
  - дочерние дополняющие разрешения:
    - условное разрешение на просмотр своих данных
      - условие: текущий пользователь и пользователь данных должны совпадать
    - условное разрешение на просмотр данных группы пользователей
      - условие: группа текущего пользователя должна совпадать с группой пользователей данных.

Дополняющие разрешения для пользователя в интерфейсе не должны никак отличаться от обычных, поэтому в конфиге они описываются так же.
Но дополняющие разрешения не являются самостоятельными.
Действию, которое они разрешают, напрямую сопоставлено их родительское Разрешение. И в коде должно проверяться именно оно, с передачей в контексте всех необходимых данных.
